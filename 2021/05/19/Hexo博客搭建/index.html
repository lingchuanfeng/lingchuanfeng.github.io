检验

启动cmd

  1. node.js 的检查

    node -v

  2. npm的检查

    npm -v

  3. Git 的检查

    git --version

以上有显示版本号，则说明安装成功

HEXO的安装

假设下边操作是在D；盘完成

  1. 在D盘启动cmd
  2. 安装之前可以先设置一下淘宝镜像加速器

    npm install -g cnpm --registry=https://registry.npm.taobao.org

  3. 全局安装框架

    npm install hexo-cli -g

  4. 创建你的博客目录

    hexo init 你博客的文件夹名字,与github名字保持一致

  5. 进入你博客的目录

    cd 你博客的文件夹名字

  6. 复制文件到你博客的目录

    npm install

  7. 安装Hexo部署插件（博客目录下启动）

    npm install hexo-deployer-git --save

GIT的配置

继续在cmd中运行

  1. 设置用户名称

    git config --global user.name "用户名"

  2. 设置用户邮箱

    git config --global user.email "用户邮箱"

  3. 生成密钥

    ssh-keygen -t rsa -C "用户邮箱"

接下来会让你设置密码，推荐全都不要设置，直接连按三次回车键。

注册登录GitHub

  1. 新建仓库，这里不多做叙述
    仓库名字为：

    “github的名字”.github.io

    然后完成创建
  2. 加入SSH
    在settings里找到SSH and GPG keys
    去new SSH key加入ssh，ssh的位置在

    C:\用户\你的win名\.ssh

    名字叫id_rsa.pub的文件，右键记事本打开，把文件复制，放到ssh上的Key里。（Title可以随便取名，一般取名blog）

博客文件配置

这一步的目的是为了能让博客部署到GitHub上，这样浏览别人也都可以浏览你的博客啦

deploy:
type: git
repo: git@github.com:你GitHub名字/你GitHub名字.github.io.git #替换成你自己仓库的HTTP URL地址
branch: master

打开你博客根目录的 _config.yml 文件，找到文件里deploy的地方（最下边）仿照上边的信息添加到里面去。

repo位置是在你创建的仓库那，code下有一个SSH链接，复制放上去。

输入：

hexo cl && hexo g && hexo d

部署ok的话就可以在浏览器输入自己的网址查看啦

HEXO常用代码解释

  1. 清理缓存

    hexo cl

    hexo cl 是 hexo clean 的简写

  2. 生成静态页面

    hexo g

    hexo g 是 hexo generate 的简写

  3. 在本地映射（预览）

    hexo s

    hexo s 是 hexo server 的简写

  4. 部署推送

    hexo d

    hexo d 是 hexo deploy 的简写

  5. 以上连写示例一（清理缓存 + 生成静态页面 + 在本地预览）

    hexo cl && hexo g && hexo s

  6. 以上连写示例二（清理缓存 + 生成静态页面 + 部署推送）

    hexo cl && hexo g && hexo d
